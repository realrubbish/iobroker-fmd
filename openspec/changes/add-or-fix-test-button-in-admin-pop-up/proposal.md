## Why

The `Test Connection` button on the adapter's wrench pop-up is currently
implemented in `src-admin/App.tsx` as a custom React button rendered
*outside* the `JsonConfig` component (the `add-test-connection-button`
change, 2026-05-31). That implementation assumed ioBroker.admin ≥ 7.7.22
loads the adapter's `admin/index.html` in an iframe, where `App.tsx`
runs and renders the button.

The 2026-06-11 E2E test session proved that assumption wrong: on the
target stack (ioBroker.admin 7.7.22, js-controller 7.1.2), the wrench
pop-up renders ioBroker's **native** `jsonConfig.json5` form, not the
Vite SPA. The custom `Test Connection` button, the `0_userdata.0.FindMyDevice`
live panel, and the `App.tsx`-managed layout never appear in the pop-up
they were designed for. The 12-second timeout, the `err.lc` change, and
the `hasValidTokens` cache from the `cleanup-bugs-and-admin-robustness`
change are all gated behind the iframe path and are therefore
unreachable from the admin UI on the deployed admin version.

The original capability requirement (a `Test Connection` button that
invokes `sendTo("testConnection", ...)` and shows the result inline
below the `Last Error` field) is still valid and was *already
documented* in `openspec/specs/admin-ui/spec.md` as a `type: "sendTo"`
jsonConfig form item. The current App.tsx implementation is a
divergence from that spec. This change brings the implementation back
in line with the spec by adding the `type: "sendTo"` item to
`src-admin/schema.json5` (so the native form gets the button) while
keeping the App.tsx button as a fallback for users on a future admin
version that does take the iframe path.

## What Changes

- Add a `type: "sendTo"` jsonConfig form item to
  `src-admin/schema.json5` for `Test Connection`. The item's
  `command` is `testConnection` and its `label` is `Test Connection`.
  This is the path ioBroker.admin renders in the native jsonConfig
  form (the surface that users actually see on admin 7.7.22).
- Keep the existing `App.tsx` button (and the 12s `Promise.race`
  timeout from `cleanup-bugs-and-admin-robustness`) as a no-op when
  the iframe path is active and the form is rendering inside `App.tsx`,
  and as the only visible path when the future admin version takes the
  iframe branch. We do not remove the App.tsx button; we stop
  double-rendering the same control.
- Make the `Last Test Result` line a `type: "staticText"` field in
  the same way the existing spec calls for: the `sendTo` reply is
  rendered by ioBroker's built-in `ConfigSendTo` widget. The
  `App.tsx`-side formatting (with the `HH:MM:SS` timestamp suffix) is
  retained for the iframe path.
- Update the admin-ui.md "Known limitation" block to reflect that the
  `Test Connection` button is now reachable in the native form, and
  remove the workaround that pointed the user at the standalone SPA
  URL.
- Update the test/verify flow in `docs/docker-development.md` to use
  the in-pop-up button rather than the standalone URL.

## Capabilities

### New Capabilities

(none)

### Modified Capabilities

- `admin-ui`: the implementation diverged from the documented
  requirement (Test Connection as a `type: "sendTo"` item in the
  schema, not a custom React button outside the `JsonConfig`
  component). This change brings the implementation back in line.
  Scenarios for "button present" and "Last test result visible" stay
  the same; the "is the button rendered inside the form the user
  actually sees" precondition changes from "admin ≥ 7.7.22 takes the
  iframe path" to "admin 7.7.22 with the native form" (and the
  App.tsx path remains for future admin versions).
- `admin-ui-delivery`: the "Wrench pop-up serves the adapter's
  `admin/index.html`" assertion in the existing spec is **not** part
  of this change. It stays as a stretch goal tracked in the existing
  investigation doc. The `admin-ui-delivery` spec is not modified
  here; this change makes the adapter functional *despite* the
  delivery gap.

## Impact

- `src-admin/schema.json5` — add one `type: "sendTo"` item
- `src-admin/App.tsx` — guard the custom button so it does not
  double-render when the schema item is present (or, more
  pragmatically, leave it in place and let the visible form decide;
  the trade-off is documented in design.md)
- `admin/jsonConfig.json5` — regenerated by `npm run build:admin`
- `admin/index.html`, `admin/index_m.html`, `admin/assets/*.js` —
  rebuilt by `npm run build:admin`
- `docs/admin-ui.md` — remove the "test via standalone URL"
  workaround; update Known limitation to point at the in-form button
- `docs/docker-development.md` — update the verify step to use the
  in-pop-up button
- `openspec/specs/admin-ui/spec.md` — add a delta clarifying that the
  `Test Connection` button is reachable in the native form via the
  `type: "sendTo"` form item

No new dependencies. No adapter-runtime changes. No public API
changes.
